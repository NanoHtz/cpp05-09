# CPP08 — Templated containers, iterators, algorithms

## De que va el modulo

El modulo 08 introduce el uso de la **STL** (Standard Template Library):

| Concepto | Descripcion |
|----------|-------------|
| **Contenedores** | `vector`, `list`, `deque`, `stack`, `map`... estructuras de datos genéricas |
| **Iteradores** | Punteros abstractos que permiten recorrer contenedores de forma uniforme |
| **Algoritmos** | Funciones de `<algorithm>` como `std::find`, `std::sort`, `std::min_element`... |

- **vector**: secuencia contigua en memoria. Acceso aleatorio O(1), pero insertar al inicio cuesta O(n) porque desplaza todos los elementos.
- **list**: lista doblemente enlazada. Inserción y eliminación O(1) en cualquier posición, pero sin acceso aleatorio.
- **deque**: bloques de memoria encadenados (como vagones de tren). Permite añadir al inicio y al final en O(1) amortizado.
- **`end()`**: apunta a la posición **después del último elemento** — nunca se desreferencia. Es la convención STL para indicar "no encontrado" o "fin del rango".

La norma del modulo obliga a **usar STL** aunque los ejercicios se puedan resolver sin ella.

# EX00 — EasyFind

## Que hace

Implementa una **función template** `easyfind` que busca un entero dentro de cualquier contenedor STL de secuencia (vector, list, deque, etc.).

---

## easyfind.hpp — Explicacion detallada

```cpp
#ifndef EASYFIND_HPP
# define EASYFIND_HPP

# include <algorithm>
# include <exception>

template<typename T>
typename T::const_iterator easyfind(const T &data, int look_for)
{
    typename T::const_iterator it = std::find(data.begin(), data.end(), look_for);
    if (it == data.end())
        throw std::exception();
    return it;
}

#endif
```

### Include guards
```cpp
#ifndef EASYFIND_HPP
#define EASYFIND_HPP
```
Evitan que el header se incluya mas de una vez en la misma compilacion.

---

### `#include <algorithm>`
Necesario para usar `std::find`. Sin esto no compila.

---

### `#include <exception>`
Necesario para usar `std::exception`. Sin esto el `throw std::exception()` no compila.

---

### `template<typename T>`
Le dice al compilador que `T` es un tipo generico. Cuando llamas `easyfind(vec, 1)`, el compilador deduce automaticamente que `T = std::vector<int>`.

---

### `typename T::const_iterator`
Este es el tipo de retorno. Como `T` es un tipo template, el compilador no sabe de antemano si `T::const_iterator` es un tipo o una variable estatica, por eso **se necesita `typename`** para aclarar que es un tipo.

Se usa `const_iterator` porque el contenedor se recibe como `const T &` — no se puede modificar, por lo que el iterador tampoco puede hacerlo.

Sin `typename` → error de compilacion.

---

### `const T &data`
Recibe el contenedor **por referencia constante**: no se copia (puede ser grande) y no se modifica.

---

### `std::find(data.begin(), data.end(), look_for)`
Recorre el contenedor de inicio a fin buscando `look_for`. Retorna un iterador al primer elemento igual a `look_for`, o `data.end()` si no lo encuentra.

---

### `if (it == data.end())`
`data.end()` es un iterador que apunta **despues del ultimo elemento** — es la convencion STL para "no encontrado". Si `find` llego al final sin exito, lanza excepcion.

---

### `return it`
Si se encontro, retorna el iterador. El llamador puede usar `*it` para obtener el valor.

---

## main.cpp — Explicacion detallada

```cpp
#include <iostream>
#include <vector>
#include <list>
#include <deque>
#include "easyfind.hpp"
```
Incluye los contenedores STL que se van a usar y el header con la funcion template.

---

### Prueba 1 — Busqueda exitosa en vector
```cpp
std::vector<int> vec;
vec.push_back(1);
vec.push_back(2);
vec.push_back(3);
vec.push_back(4);

std::vector<int>::const_iterator it = easyfind(vec, 1);
std::cout << "Found in vector: " << *it << std::endl;
```
Crea un vector con los valores `{1, 2, 3, 4}` y busca el `1`.
`easyfind` lo encuentra y devuelve un const_iterator. Se desreferencia con `*it` para imprimir el valor.
**Salida:** `Found in vector: 1`

---

### Prueba 2 — Busqueda fallida en vector
```cpp
easyfind(vec, 42);
```
Busca el `42` en el mismo vector. Como no existe, `easyfind` lanza `std::exception`.
El `catch` lo atrapa e imprime el mensaje de error.
**Salida:** `42 not found in vector`

---

### Prueba 3 — Busqueda exitosa en list
```cpp
std::list<int> lst;
lst.push_back(10);
lst.push_back(20);
lst.push_back(30);

std::list<int>::const_iterator it = easyfind(lst, 20);
std::cout << "Found in list: " << *it << std::endl;
```
Demuestra que `easyfind` funciona igual con `std::list`. El tipo del iterador cambia
(`std::list<int>::const_iterator`) pero la logica es identica.
**Salida:** `Found in list: 20`

## Flujo del main

| Prueba | Contenedor | Valor buscado | Resultado esperado |
|--------|-----------|--------------|-------------------|
| 1 | `vector {1,2,3,4}` | `1` | Found in vector: 1 |
| 2 | `vector {1,2,3,4}` | `42` | 42 not found in vector |
| 3 | `list {10,20,30}` | `20` | Found in list: 20 |
| 4 | `deque {10,20,30}` | `20` | Found in deque: 20 |

---

## Resumen del flujo

```
easyfind(contenedor, valor)
    → std::find busca linealmente
    → si no encuentra → throw exception
    → si encuentra    → return const_iterator
```

## Por que no hay easyfind.cpp

Las **funciones template** deben estar completamente definidas en el header (`.hpp`), no en un `.cpp`.

El motivo: cuando el compilador encuentra `easyfind(vec, 1)`, necesita generar el codigo de la funcion para `T = std::vector<int>`. Para eso necesita ver la implementacion completa en el momento de la instanciacion — si estuviera en un `.cpp` separado, el compilador no la veria y daria error de enlazado.

Regla general en C++98:
- Funciones normales → declaracion en `.hpp`, implementacion en `.cpp`
- Funciones template → todo en el `.hpp` (o en un `.tpp` incluido dentro del `.hpp`)

---

## Puntos clave

- **Template** → funciona con cualquier contenedor que tenga `.begin()`, `.end()` y `::const_iterator`
- **`std::find`** hace la búsqueda lineal O(n)
- Usa **excepciones** para indicar "no encontrado" en vez de retornar un valor especial
- Solo funciona con contenedores de `int` (el segundo parámetro está hardcodeado como `int`)
- No funciona con contenedores asociativos (`std::map`, `std::set`) porque no tienen una secuencia lineal de valores directamente buscable con `std::find`
- No hay `easyfind.cpp` — las funciones template van enteras en el header
