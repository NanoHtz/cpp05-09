# CPP08 — Templated containers, iterators, algorithms

## De que va el modulo

El modulo 08 introduce el uso de la **STL** (Standard Template Library):

| Concepto | Descripcion |
|----------|-------------|
| **Contenedores** | `vector`, `list`, `deque`, `stack`, `map`... estructuras de datos genéricas |
| **Iteradores** | Punteros abstractos que permiten recorrer contenedores de forma uniforme |
| **Algoritmos** | Funciones de `<algorithm>` como `std::find`, `std::sort`, `std::min_element`... |

Un vector es una sucesion de datos, le cuesta añadir datos al inicio, pues tiene que mover todo.
Una list, una lista doblemente enlazada.
Un deque, son bloques de memoria, como vagones de un tren, puedes añadir memoria antes y despues.
data.end() -> apunta a un ultimo elemento que en realidad no existe, asique si buscas algo que esta en el ultimo elemento no saldra error.
usamos const.

La norma del modulo obliga a **usar STL** aunque los ejercicios se puedan resolver sin ella.

# EX00 — EasyFind

## Que hace

Implementa una **función template** `easyfind` que busca un entero dentro de cualquier contenedor STL de secuencia (vector, list, deque, etc.).

---

## EasyFind.hpp — Explicacion detallada

```cpp
#ifndef EASYFIND_HPP
# define EASYFIND_HPP

# include <algorithm>

template<typename T>
typename T::iterator easyfind(T &a, int b)
{
    typename T::iterator it = std::find(a.begin(), a.end(), b);
    if (it == a.end())
        throw std::exception();
    return it;
}

#endif
```

### Include guards
```cpp
#ifndef EASYFIND_HPP
#define EASYFIND_HPP
```
Evitan que el header se incluya mas de una vez en la misma compilacion.

---

### `#include <algorithm>`
Necesario para usar `std::find`. Sin esto no compila.

---

### `template<typename T>`
Le dice al compilador que `T` es un tipo generico. Cuando llamas `easyfind(vec, 3)`, el compilador deduce automaticamente que `T = std::vector<int>`.

---

### `typename T::iterator`
Este es el tipo de retorno. Como `T` es un tipo template, el compilador no sabe de antemano si `T::iterator` es un tipo o una variable estatica, por eso **se necesita `typename`** para aclarar que es un tipo.

Sin `typename` → error de compilacion.

---

### `T &a`
Recibe el contenedor **por referencia** para no copiarlo (puede ser grande).

---

### `std::find(a.begin(), a.end(), b)`
Recorre el contenedor de inicio a fin buscando `b`. Retorna un iterador al primer elemento igual a `b`, o `a.end()` si no lo encuentra.

---

### `if (it == a.end())`
`a.end()` es un iterador que apunta **despues del ultimo elemento** — es la convencion STL para "no encontrado". Si `find` llego al final sin exito, lanza excepcion.

---

### `return it`
Si se encontro, retorna el iterador. El llamador puede usar `*it` para obtener el valor.

---

## main.cpp — Explicacion detallada

```cpp
#include <iostream>
#include <vector>
#include <list>
#include <deque>
#include "Easyfind.hpp"
```
Incluye los contenedores STL que se van a usar y el header con la funcion template.

---

### Prueba 1 — Busqueda exitosa en vector
```cpp
std::vector<int> vec;
vec.push_back(1);
vec.push_back(2);
vec.push_back(3);
vec.push_back(4);

std::vector<int>::iterator it = easyfind(vec, 3);
std::cout << "Found in vector: " << *it << std::endl;
```
Crea un vector con los valores `{1, 2, 3, 4}` y busca el `3`.
`easyfind` lo encuentra y devuelve un iterador. Se desreferencia con `*it` para imprimir el valor.
**Salida:** `Found in vector: 3`

---

### Prueba 2 — Busqueda fallida en vector
```cpp
easyfind(vec, 42);
```
Busca el `42` en el mismo vector. Como no existe, `easyfind` lanza `std::exception`.
El `catch` lo atrapa e imprime el mensaje de error.
**Salida:** `42 not found in vector`

---

### Prueba 3 — Busqueda exitosa en list
```cpp
std::list<int> lst;
lst.push_back(10);
lst.push_back(20);
lst.push_back(30);

std::list<int>::iterator it = easyfind(lst, 20);
std::cout << "Found in list: " << *it << std::endl;
```
Demuestra que `easyfind` funciona igual con `std::list`. El tipo del iterador cambia
(`std::list<int>::iterator`) pero la logica es identica.
**Salida:** `Found in list: 20`

## Flujo del main

| Prueba | Contenedor | Valor buscado | Resultado esperado |
|--------|-----------|--------------|-------------------|
| 1 | `vector {1,2,3,4}` | `3` | Found in vector: 3 |
| 2 | `vector {1,2,3,4}` | `42` | 42 not found in vector |
| 3 | `list {10,20,30}` | `20` | Found in list: 20 |

---

## Resumen del flujo

```
easyfind(contenedor, valor)
    → std::find busca linealmente
    → si no encuentra → throw exception
    → si encuentra    → return iterador
```

## Puntos clave

- **Template** → funciona con cualquier contenedor que tenga `.begin()`, `.end()` y `::iterator`
- **`std::find`** hace la búsqueda lineal O(n)
- Usa **excepciones** para indicar "no encontrado" en vez de retornar un valor especial
- Solo funciona con contenedores de `int` (el segundo parámetro está hardcodeado como `int`)
